# dotfiles

Personal Mac setup for DevOps work. Covers everything from a fresh macOS install to a fully configured environment.

## Structure

```
dotfiles/
├── Brewfile              # All packages and apps (Homebrew)
├── setup.sh              # Fresh install script
├── git/.gitconfig        # Git config template
├── zsh/.zshrc            # Zsh config and aliases
├── scripts/
│   └── setup-ssh.sh      # Configures git signing key per device
└── docs/
    └── setup.md          # Detailed setup guide
```

## Quick start

```bash
git clone https://github.com/joaovitor/dotfiles ~/dotfiles
cd ~/dotfiles
bash setup.sh
source ~/.zshrc
```

`setup.sh` installs Homebrew, all packages from `Brewfile`, and configures git and zsh. It will ask for your SSH signing key during setup.

## Highlights

- **Bitwarden** (self-hosted via Vaultwarden) manages passwords and SSH keys via SSH Agent
- **Two SSH keys per device** — one for authentication, one for signing Git commits
- **Git commits are signed** automatically via SSH
- **mise** manages runtime versions (Node, Python, Terraform) per project via `.mise.toml`
- **Podman** instead of Docker
- **SimpleLogin** (via Proton) for email aliases, integrated with Bitwarden

## Adding packages

```bash
brew install <tool>              # then add: brew "<tool>" to Brewfile
brew install --cask <app>        # then add: cask "<app>" to Brewfile
```

Runtime versions go in `.mise.toml` inside each project, not in Brewfile.

## Per-device setup

Some config is device-specific and lives outside the repo:

| File | Content |
|---|---|
| `~/.gitconfig` | Generated by `setup-ssh.sh` with the device's signing key |
| `~/.ssh/allowed_signers` | Public signing key for local commit verification |
| `~/.bitwarden-ssh-agent.sock` | Created by Bitwarden at runtime |

See [docs/setup.md](docs/setup.md) for the full guide.
